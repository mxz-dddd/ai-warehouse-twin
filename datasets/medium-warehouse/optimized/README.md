# Medium Warehouse Optimized Slotting

This directory contains the A5a/A5b R3 Phase-1 demo ABC slotting output for
`datasets/medium-warehouse/scenario.json`.

`abc-slotting-config.json` is generated by a deterministic demo heuristic:

- SKU demand is ranked by total ordered quantity across outbound and each-pick
  orders, then by order line count, then by SKU id.
- Candidate pick locations are ranked by modeled distance to the A-zone pick
  path anchor `aisle-a-main`, using the existing layout graph and shortest-path
  routing.
- Assignments pair the highest-ranked SKUs with the highest-ranked candidate
  pick locations.

`scenario.json` applies those assignments to a derived optimized scenario.
`artifacts/run-artifact.v1.json` and `artifacts/movement-artifact.v1.json` are
the deterministic modeled optimized artifacts generated from that derived
scenario. `artifacts/comparison-artifact.v1.json` compares the committed A4b
baseline RunArtifact with the optimized RunArtifact and includes `kpi_deltas`
and `improvement_pct`.

This is a deterministic demo ABC slotting heuristic for R3 Phase-1. It is not a
globally optimal slotting solution. Results are deterministic modeled
simulation artifacts, not calibrated WMS or sensor-ground-truth measurements.
In the current core timing model, the optimized scenario does not improve the
tracked KPI values; the A5b ComparisonArtifact records this honestly with 0%
`improvement_pct` values rather than inventing a gain.

The baseline scenario and A4b baseline golden artifacts remain unchanged.
